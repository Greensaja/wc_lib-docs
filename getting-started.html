Getting started

Install wc_lib

Three steps: drop the resource in, ensure it before anything that depends on it, optionally pin the framework. That's the whole setup.

1. Add the resource

Copy the wc_lib folder into your server's resources directory, alongside your other resources — not nested inside one of them.

file tree
resources/
├── [core]/
│   └── vorp_core/
├── [standalone]/
│   └── wc_lib/              ← here
└── [wc]/
    ├── wc_encounter/
    └── wc_death/

2. Ensure it before dependent resources

wc_lib needs to start before any resource that calls its exports. Add it to your server.cfg ahead of everything else with a "wc_" prefix or anything else you plan to migrate.

server.cfg 
ensure vorp_core
ensure wc_lib
ensure wc_encounter
ensure wc_death
Order matters. If a resource calling exports.wc_lib:GetMoney(...) starts before wc_lib has finished initializing, the export won't exist yet. Keep ensure wc_lib directly after your core framework and before anything that uses it.

3. Confirm it picked up your framework

On start, wc_lib auto-detects which core is running by checking for vorp_core or rsg-core as a started resource. Check your server console for a line like:

console output
[wc_lib] server ready — framework: vorp — version: 1.0.0
[wc_lib] client ready — framework: vorp — version: 1.0.0

If you see NONE DETECTED instead, your core framework either isn't installed or started after wc_lib. Check your ensure order, or set the framework explicitly — see below.

Optional: force the framework

Auto-detection covers almost every setup. The one case it doesn't is running both cores side by side mid-migration — wc_lib will default to VORP and print a warning if it sees both. Override it explicitly in shared/config.lua:

wc_lib/shared/config.lua 
-- leave as nil to auto-detect (the default)
WCLibConfig.ForceFramework = 'vorp' -- or 'rsg'

Calling wc_lib from your own resource

Every function is exposed as a standard FXServer export. No require, no import — just call it the way you'd call any other resource's export.

server-side, any resource 
local money = exports.wc_lib:GetMoney(source)
exports.wc_lib:AddMoney(source, 50)
exports.wc_lib:Notify(source, { title = "+$50" })
client-side, any resource 
local prompt = exports.wc_lib:CreatePrompt("Talk [G]")
exports.wc_lib:SetPromptVisible(prompt, true)
Tip. If you want autocomplete-friendly access inside wc_lib itself, the full WCLib table is also available as a global within wc_lib's own files — but from outside the resource, always go through exports.wc_lib:FunctionName(...).
